# LinkedIn Reply Frameworks

## Framework by Reply Category

### 1. INTERESTED — Lead wants to engage

**Framework: Acknowledge → Micro-Value → Soft CTA**

Lead subtypes:
- Wants demo/walkthrough → send calendar + mention personalized demo
- Asks about pricing → share framework, redirect to call
- Wants materials/info → send + personalized note + follow-up in 2-3 days
- Positive general ("sounds good") → propose meeting immediately
- Redirects to colleague → thank, reach out to new person with referral mention
- Wants trial/test → set up trial + guided onboarding

**Priority:** Respond within 2 hours.

**Example (EN):**
```
Thanks, {name}! Quick context: agencies like {similar_company} use our white-label API to keep creator data under their brand - most integrations go live in under 2 weeks.

Worth a 15-min call to see if it maps to what {company} needs? Happy to run a quick demo on any creator handle.
```

**Example (RU):**
```
{name}, спасибо за ответ! Коротко: агентства вроде {similar_company} используют наш API чтобы отдавать клиентам данные по креаторам под своим брендом - интеграция занимает пару недель.

Можем созвониться на 15 минут - покажу как это работает для {company}?
```

### 2. MEETING_REQUEST — Lead wants to meet

**Framework: Confirm → Add Value → Lock Time**

**Priority:** Respond within 1 hour.

- If specific time proposed → confirm immediately
- If asks for calendar → send link, done
- If generic "let's talk" → calendar link + 2 time slots

**Meeting agenda to outline (adapt each point to the lead's company):**

1. **Learn about them first** — we'll read up on their company beforehand, but on the call we confirm: are they a platform, agency, or another type? What data do they currently use/need? Adapt this point to what's known (e.g. for an agency: "understand which verticals your clients are in and what creator data you report on"; for a platform: "see where our API fits into your existing stack").
2. **Live platform demo** — walk through OnSocial's APIs and product, tailored to their use case. Show real creator data relevant to their market/niche. Not a generic demo - pick creators or segments that matter to THEM.
3. **Pricing discussion** — based on what they outline on the call (volume, endpoints, use case), or a general overview of how pricing works if they're still exploring.
4. **Free trial + next steps** — offer a trial, agree on follow-up over email.

When writing the reply, translate these 4 points into natural language adapted to the specific lead. Don't use numbered lists with generic language - make each point feel like it's about THEIR company.

**Example (generic):**
```
{name}, great - let's do it!

On the call I'll walk through a few things with {company} in mind:
- {personalized point about understanding their setup}
- Live demo on {relevant creators/data for their market}
- Pricing based on what you're looking for
- If it's a fit, we can set up a free trial right after

Here's my calendar: {link}
```

**Example (for an Indian IM agency like Flutch):**
```
{name}, great - let's do it.

On the call I'll walk you through a few things tailored to {company}:
- How our creator data maps to the Indian market and verticals you cover
- Live API demo - bring a few creator handles and I'll run them through the system
- Pricing based on your volume and use case
- If it clicks, we set up a free trial and take it from there on email

Here's my calendar: {link}
```

**Example (for a SaaS platform like CreatorIQ):**
```
{name}, sounds good.

Here's what I'm thinking for the call:
- Quick look at where our data API could plug into {company}'s stack
- Live demo on the endpoints most relevant to you - audience demographics, fraud scoring, brand affinity
- Walk through pricing based on your expected call volume
- If it makes sense, spin up a trial so your team can test the integration

Calendar: {link}
```

### 3. QUESTION — Lead has questions

**Framework: Answer → Proof → Invite Deeper**

**Priority:** Respond within 4 hours.

Subtypes:
- Pricing → share framework (NOT exact numbers), redirect to call
- How it works → 2-3 sentence answer + offer demo
- Technical/API → specific answer with data points + API docs link
- Competitor comparison → never badmouth, focus on differentiators, offer side-by-side
- Coverage/platform → share specific numbers

**Example:**
```
Great question - short answer: we cover 450M+ profiles across Instagram, TikTok, and YouTube. Audience demographics go down to city level, plus fraud detection and brand affinity data.

Happy to run any creator handle through the system so you can see the output format. Want me to?
```

### 4. NOT_INTERESTED — Graceful exit

**Framework: Acknowledge → Leave Door Open**

**Priority:** Low.

Subtypes:
- Has existing solution → respect, mention availability to compare
- Polite decline → thank, wish well, no follow-up
- Aggressive decline / spam → remove from all campaigns, DON'T reply
- Not relevant → thank, update ICP data

**Example (EN):**
```
Totally understand, {name}. Appreciate you letting me know.

If {company} ever needs creator data under your own brand - we'll be here. Best of luck!
```

**Example (RU):**
```
{name}, понял, спасибо за честный ответ. Если что-то изменится - всегда на связи. Удачи!
```

### 5. OUT_OF_OFFICE

**Framework: Note Return Date → Schedule Follow-up**

- Note return date
- Schedule follow-up for return date + 2 days
- If colleague mentioned for urgent matters → contact if relevant

### 6. WRONG_PERSON

**Framework: Thank → Get Right Contact → Reach Out**

- Left company → mark in CRM, find replacement
- Redirects to another → thank, email the suggested person with referral mention

**Example:**
```
Thanks for pointing me in the right direction, {name}! Could you share their email or LinkedIn? I'll mention you reached out.
```

### 7. REDIRECT / REFERRAL

When lead says "talk to X instead":

```
{name}, thanks for the intro! I'll reach out to {referred_name} and mention you connected us.

Appreciate the help.
```

## Reply Frameworks (Industry Best Practice)

### Framework A: Problem → Impact → Proof → Soft Ask
Best for: interested & skeptical replies
- Restate/name the problem briefly
- Explain business impact in one line
- One quick example or metric
- Low-pressure next step

### Framework B: Acknowledge → Answer → Add Value → Invite
Best for: direct questions
- Acknowledge the question
- Give concise answer
- Add a small resource or example
- Invite follow-up

### Framework C: Validate → Reframe → Lower-Commitment Option
Best for: objections (budget, timing, existing solution)
- Validate the objection
- Reframe with how others overcame it
- Offer pilot, data share, or intro

### Framework D: Respectful Close + Future Permission
Best for: not interested
- Thank for time
- Offer to stay in touch with occasional insights
- Note specific area you'll keep on radar
