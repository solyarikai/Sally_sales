# Anti-Patterns — What NOT to Do

These patterns were observed in dismissed/rejected draft replies. NEVER use them.

## Corporate Robot Language
- "I hope this message finds you well"
- "I would be happy to share more information about our services"
- "We appreciate your inquiry and would be delighted to provide..."
- "If you have any specific questions or if there's anything I can assist you with"
- "Looking forward to hearing from you soon"
- "Please let me know your availability, and we can set something up"
- "Thank you for reaching out and expressing interest in our products and services"
- "Best regards, [Your Name] [Your Position] [Your Company] [Your Contact Information]"

## Structural Problems
- Responding to empty/blank emails with full pitch
- Adding signatures with placeholder brackets [Your Name]
- Using formal sign-offs in LinkedIn DMs (Best regards, Kind regards)
- Writing 200+ word replies to 1-sentence messages
- Repeating the same pitch across follow-ups without new value
- Pitching immediately after connection accept (looks like bot)
- Sending calendar links before the lead asked for a call

## Tone Mistakes
- Overly formal when lead is casual
- Feature-dump paragraphs
- Starting with "Dear" in LinkedIn DMs
- Using "we" exclusively instead of "I" (feels corporate)
- Buzzword-heavy sentences that read like marketing copy
- Asking multiple questions in one message
- "Worth 20 minutes?" — zaезженный cold outreach шаблон, считывается как скрипт; для тёплого диалога (lead уже заинтересован) использовать конкретный оффер: "Want to hop on a call and I'll run [market] live?" или "Should we find a time this week?"

## Operational Mistakes
- Replying to "STOP" or unsubscribe requests
- Following up on aggressive declines
- Not matching the lead's language
- Sending the same message text as initial outreach in a reply
- Not acknowledging what the lead actually said
- Ignoring specific questions to push agenda

## Formatting Mistakes (LinkedIn specific)
- Em dashes (—) — break in some clients
- HTML tags in message body
- Long unbroken paragraphs (no line breaks)
- Multiple links in one message
- Excessive emoji usage

## Humanizer: 29 AI Writing Patterns to Avoid

Based on blader/humanizer (Wikipedia's "Signs of AI writing" guide).
These patterns MUST be checked in every generated reply.

### Content Patterns
1. **Significance inflation** — "pivotal moment", "testament to", "vital role", "enduring legacy" → state facts plainly
2. **Notability name-dropping** — listing media outlets without context → cite specific instance
3. **Superficial -ing analyses** — "highlighting...", "showcasing...", "reflecting..." → remove or expand with facts
4. **Promotional language** — "vibrant", "groundbreaking", "nestled", "breathtaking", "renowned" → neutral description
5. **Vague attributions** — "Experts believe", "Industry observers note" → name the source or remove
6. **Formulaic challenges** — "Despite challenges... continues to thrive" → specific facts

### Language Patterns
7. **AI vocabulary** — additionally, crucial, delve, fostering, landscape, pivotal, showcase, tapestry, testament, underscore, vibrant → use plain words
8. **Copula avoidance** — "serves as", "stands as", "features", "boasts" → use "is", "has"
9. **Negative parallelisms** — "It's not just X, it's Y" → state the point directly
10. **Rule of three** — "innovation, inspiration, and insights" → use natural number of items
11. **Synonym cycling** — switching between "tool/solution/platform/system" → pick one and repeat
12. **False ranges** — "from X to Y" where not a real scale → list items directly
13. **Passive voice** — "No configuration needed" → name the actor

### Style Patterns
14. **Em dash overuse** — prefer commas or periods
15. **Boldface overuse** — don't bold every keyword
16. **Inline-header lists** — "**Speed:** Speed improved" → convert to prose
17. **Title Case in headings** — use sentence case
18. **Emojis as decoration** — remove unless lead uses them
19. **Curly quotes** — use straight quotes
26. **Hyphenated pairs overuse** — "cross-functional, data-driven" → drop hyphens on common pairs
27. **Persuasive authority tropes** — "At its core", "The real question is" → state point directly
28. **Signposting** — "Let's dive in", "Here's what you need to know" → start with content
29. **Fragmented headers** — heading + one-line restatement → let heading do the work

### Communication Patterns
20. **Chatbot artifacts** — "I hope this helps!", "Let me know if..." → remove entirely
21. **Cutoff disclaimers** — "While details are limited..." → find facts or remove
22. **Sycophantic tone** — "Great question!", "You're absolutely right!" → respond directly

### Filler and Hedging
23. **Filler phrases** — "In order to" → "To"; "Due to the fact" → "Because"; "It is important to note" → remove
24. **Excessive hedging** — "could potentially possibly" → "may"
25. **Generic conclusions** — "The future looks bright" → specific next step or fact
