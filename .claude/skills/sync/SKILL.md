# Скилл: /sync — универсальный синк после сессии

## Цель

Зафиксировать результаты рабочей сессии: обновить память и документацию.
Работает с **любым проектом** — определяет контекст автоматически.

---

## Шаг 0 — Определи проект (автоматически)

**Не задавай вопросов.** Определи всё сам:

1. **Проект** = имя из текущей рабочей директории (cwd)
2. **CLAUDE.md** — прочитай `CLAUDE.md` в корне проекта (если есть) для понимания структуры
3. **MEMORY.md** — найди файл памяти. Проверь оба варианта:
   - `~/.claude/projects/<encoded-cwd>/memory/MEMORY.md` (стандартная авто-память Claude)
   - `<cwd>/.claude/memory/MEMORY.md` (встроенная в проект)
   - Используй тот, что существует. Если оба — обнови оба.
4. **Документация** — найди через Glob в корне проекта:
   - `docs/CURRENT_STATE.md` или `CURRENT_STATE.md`
   - `docs/ROADMAP.md` или `ROADMAP.md`
   - `README.md` (для проектов без CURRENT_STATE)
   - Любые `docs/*.md` с датами/статусами
> **Ключевой принцип:** скилл адаптируется к тому, что есть.
> Нет CURRENT_STATE — пропускаем. Нет ROADMAP — пропускаем. Обновляем только то, что реально существует.

---

## Шаг 1 — Анализ сессии из истории чата

**Не задавай вопросов.** Восстанови контекст из переписки:
- Что обсуждалось и делалось в этой сессии
- Какие задачи были начаты, завершены или заблокированы
- Какие файлы были изменены
- Какие новые задачи или проблемы всплыли

---

## Шаг 2 — Читай текущее состояние (параллельно)

Одновременно читай все найденные в шаге 0 файлы:
- MEMORY.md
- CURRENT_STATE.md / README.md
- ROADMAP.md (если есть)

---

## Шаг 3 — Обнови документацию проекта

Для каждого найденного файла — обнови только то, что изменилось:

### CURRENT_STATE.md (если есть)
- Дату в заголовке → сегодняшняя
- Статусы: `⚠️` → `✅` для завершённых
- Критические баги — убери исправленные, добавь новые
- % готовности — только если есть явные основания

### README.md (если нет CURRENT_STATE)
- Отметь завершённые задачи: `- [ ]` → `- [x]`
- Добавь новые задачи если всплыли
- Обнови раздел "Материалы" / "Статус" если есть

### ROADMAP.md (только если этап завершён целиком)
- Отметь этап как `✅ Завершён`
- Если этап не завершён — не трогай

---

## Шаг 4 — Обнови MEMORY.md

Обнови только то, что реально изменилось:
- Раздел "Следующая задача" — если приоритет сменился
- Раздел "Что сделано" — добавь запись о сессии
- Раздел "Что осталось" — убери завершённое, добавь новое
- Критические баги — убери исправленные
- Текущий статус — если прогресс изменился

Если в проекте есть отдельные learnings-файлы (как `projects/[name]_learnings.md`):
- Добавь запись сессии: дата, задача, результат, паттерны

---

## Шаг 5 — Финальный отчёт

Выведи краткий итог (≤ 8 строк):

```
Синк завершён.

Docs: обновлён <файл> (<что именно>)
Memory: обновлено (<что именно>)

Следующий приоритет: <задача>
```

Если доки не обновлялись — не включай строку про них.

---

## Правила

- **Не спрашивай разрешения** — просто делай
- **Точечные правки** — Edit tool, не Write (для существующих файлов)
- **Параллельность** — читай файлы одновременно в шаге 2
- **Не придумывай задачи** — только то, что реально было в сессии
- **Консерватизм по %** — меняй только с явными основаниями
- **Если ничего не изменилось** — скажи об этом, не делай пустые правки
- **Адаптивность** — работай с тем, что есть в проекте, не требуй конкретной структуры
- **Определяй проект автоматически** — из cwd, не спрашивай
