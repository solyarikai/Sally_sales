# Скилл: translate — перевод документа на русский

## Цель

Прочитать указанный файл, перевести его содержимое на русский язык и сохранить
результат в файл `<basename>_ru.md` рядом с оригиналом.

## Процесс

### Шаг 1 — Определить файл

Путь к файлу берётся из аргумента пользователя (например, `/translate path/to/file.md`).

Если путь не указан — спросить у пользователя: "Какой файл переводим?"

### Шаг 2 — Прочитать файл

Вызови Read с абсолютным путём к файлу.

### Шаг 3 — Перевести

Переведи содержимое на русский язык, соблюдая правила:

- Сохранять markdown-структуру: заголовки (`#`, `##`), списки, **жирный**, *курсив*, таблицы
- Профессиональные термины не переводить: DM, NOC, SQL, UHNW, MEP, DDA, Trakhees, DCD, TAM, CAGR, GTM, MVP, PRD, ROI, B2B, SaaS и аналогичные
- Ссылки и URL оставлять без изменений
- Не добавлять комментарии, пояснения или от себя — только перевод
- Имена собственные (компании, продукты, бренды) не переводить

### Шаг 4 — Сохранить

Определи имя выходного файла: взять basename без расширения, добавить `_ru.md`.

Примеры:
- `Context.md` → `Context_ru.md`
- `You_grow.md` → `You_grow_ru.md`
- `guide_en.md` → `guide_en_ru.md`

Сохрани файл в ту же директорию, что и оригинал, с помощью Write.

### Шаг 5 — Финальный отчёт

```
Переведено: <путь к _ru.md>
```

Одна строка, без лишних слов.

## Правила

- **Структура** — выходной файл должен быть идентичен оригиналу по структуре, только текст на русском
- **Термины** — не переводить технические аббревиатуры и профессиональный жаргон
- **Ссылки** — URL и markdown-ссылки оставлять как есть
- **Без добавлений** — не добавлять заголовки типа "Перевод:" или мета-информацию, если их нет в оригинале
- **Путь** — всегда сохранять рядом с оригиналом, не в CWD
