COMPLETED: Use multi-stage builds

Changes made:
- Updated backend/Dockerfile to use multi-stage builds
- Stage 1 (builder): Uses python:3.11-slim, installs build dependencies (gcc, libpq-dev), creates virtual environment, and installs Python packages
- Stage 2 (production): Uses python:3.11-slim, installs only runtime dependencies (libpq5), copies virtual environment from builder stage
- Frontend Dockerfile already had multi-stage builds (build with node:20-alpine, production with nginx:alpine)

Benefits:
- Smaller final image size (no gcc/build tools in production image)
- Better security (fewer packages = smaller attack surface)
- Faster deployments (smaller images to push/pull)
