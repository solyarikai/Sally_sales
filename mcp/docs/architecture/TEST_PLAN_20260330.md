### Layer 6: Conversation History Verification

Every MCP tool call must be logged and visible in `/conversations` page.

| Check | How | Verify |
|-------|-----|--------|
| Tool calls logged | After each tool call → query `mcp_conversation_logs` table | Row exists with correct tool_name, user_id, timestamp |
| Conversations page shows all calls | Screenshot `/conversations` after full test | Count of rows matches number of tool calls made |
| Direction correct | Each log entry | `client_to_server` for tool calls, method = `tools/call` |
| User-scoped | Login as different user → check `/conversations` | Only THIS user's calls visible, not other users' |
| Content summary readable | Each log entry | Summary contains tool name + truncated args, not garbage |
| Chronological order | All entries | Timestamps ascending, no gaps |
| Session ID tracked | All entries in one test run | Same session_id across all calls in one connection |

**Test procedure:**
1. Run full default flow (21 steps) via real MCP SSE
2. Count total tool calls made
3. Query DB: `SELECT COUNT(*) FROM mcp_conversation_logs WHERE user_id = X AND session_id = Y`
4. Assert: DB count = tool calls made
5. Screenshot `/conversations` page → verify all calls visible
6. Assert: no calls from other test users visible
