# PART 2: GOD-LEVEL FIX PLAN

## The Correct Flow (User-Approved)

```
Step 1: create_project
  → Scrape website → extract offer + target roles
  → Show: offer, value prop, target audience, target roles, PROJECT LINK
  → Ask: "Does this look correct?"
  → WAIT for approval

Step 2: confirm_offer
  → User approves offer + roles
  → Ask: "Have you launched campaigns for this segment before?"
  → WAIT (blacklist question)

Step 3: Blacklist answer
  → If yes: user provides campaign pattern → load contacts for exclusion
  → If no: skip, move to next step
  → Ask: "Which email accounts to use for the campaign?"
  → WAIT (email accounts)

Step 4: align_email_accounts
  → User says "all accounts with elnar in name" or similar
  → Show matched accounts, confirm
  → WAIT

Step 5: tam_gather PREVIEW
  → Probe Apollo (1 credit, per_page=100 — save companies)
  → Generate 20+ keywords via filter_mapper
  → A11 classifier: industry-first vs keywords-first
  → Cost estimate using ACTUAL target_count
  → CREATE pipeline in pending_approval state
  → Show in chat AND provide pipeline link:
    - Strategy: industry-first (WHY: ...)
    - Keywords: 33 generated (show all or top 20)
    - Industries: apparel & fashion, luxury goods & jewelry
    - Location: Italy, Size: 1-200
    - Available: 8,127 companies in Apollo
    - KPIs: 100 target people, max 3/company
    - Cost: ~10 pages = 10 credits + ~100 enrichment = ~$1.10
    - Pipeline: http://46.62.210.24:3000/pipeline/XXX
    - Blacklist: none / loaded from campaign X
    - Email accounts: 14 Elnar accounts selected
  → Ask: "Proceed?"
  → WAIT

Step 6: User approves
  → tam_gather CONFIRM + run_auto_pipeline in ONE step
  → Pipeline starts AUTOMATICALLY:
    - Gathers companies from Apollo (10 pages in parallel)
    - Each company → scrape → classify → people (streaming, no batches)
    - KPI checked after each person
    - When KPI met → auto-generate sequence → auto-push to SmartLead as DRAFT
  → Returns: "Pipeline started. Track at: http://..."
```
