### Проблема с текущим сбором
- Из 790 собранных в Apollo + обогащённых через Clay → Crona оставил только ~150 после сегментации
- Причина: фильтры в Apollo слишком узкие → нужно **пересмотреть фильтры**
- Стат. значимость для теста гипотезы = **минимум 500 контактов на сегмент**

### Технический стек
- Текущий пайплайн: Apollo → Clay → Crona → SmartLead
- **Следующий шаг: перейти на репозиторий** (там уже все интеграции, не нужна Crona, нет расхода кредитов)
- Crona → используй пока для прототипирования пайплайна, потом переносишь в репо

### SmartLead — роль в пайплайне и что обсуждалось

**Что SmartLead делает в текущем стеке:**
- Это финальная точка пайплайна — туда заливаются контакты, оттуда уходят письма
- Вся история коммуникаций по OnSocial живёт в SmartLead (+ GoSequence) — **это источник истины** по всем ответам и отправкам
- Для OnSocial не было WhatsApp или Telegram, как в других проектах — только SmartLead и email

**Про API SmartLead (начало созвона, 00:00:12):**
Петя упоминает это в самом начале — в контексте того, что нужно объяснить Клоду при работе с репо:
> "Есть один endpoint, который возвращает все контакты по компании. У нас уже вся логика есть в репозитории, которая тебя [к нему] имеет подключить. В целом, это быстро собирается. Чтобы проверить на предмет уникальностей. Потом GoSequence."

То есть: **в репо уже есть готовая интеграция со SmartLead API** — один вызов → все контакты по компании → проверка уникальности → GoSequence. Не нужно строить с нуля, нужно просто подключиться через Амира.

**SmartLead НЕ является центром управления (00:14:18):**
Sofia спросила: "Центр управления полетами — это SmartLead?"
Петя ответил: "Нет. Репозиторий." — SmartLead это инструмент отправки, но управление пайплайном идёт через репо.

**Аналитика и отчёты (00:25:01):**
- В SmartLead есть запущенные кампании с автоматической отправкой — с них надо снять метрику по дням
- Там можно смотреть отчёты по каждой кампании

**Шаблоны ответов Насти:**

### Полный технический стек — детально

#### Пайплайн сейчас vs. будущий

```
Сейчас:   Apollo → Clay (обогащение) → Crona (сегментация) → GoSequence → SmartLead
Будущий:  Apollo → Clay → Репо (сегментация + уникальность + парсинг) → GoSequence → SmartLead
```

#### Репозиторий — центр управления (00:14:18–00:14:32)
- SmartLead — **НЕ** центр управления. Центр — **репозиторий**
- В репо уже встроен движок со всеми touchpoints и историей коммуникаций
- Там уже всё интегрировано — открываешь код и используешь напрямую
- Работа через git: коммиты, пуши — таким образом улучшается общая система

**Конкретно что уже есть в репо (00:00:12):**
- Один endpoint → возвращает все контакты по компании (из SmartLead)
- Логика проверки уникальностей уже написана
- После проверки → передаёт в GoSequence

#### Crona — только для прототипирования (00:27:09–00:27:35)
- Crona = "чертёжник для пайплайна" — придумать логику, проверить гипотезу
- Потом всё то же самое делает репо, **без расхода кредитов**
- Репо умеет **парсить сайты** — так же как Crona, но бесплатно
- Вывод: Crona → прототип → переносишь в репо → масштабируешь

#### Кредиты (00:26:55–00:28:37)
- **Apollo** — кредиты использовать свободно, лимитов нет
- **Crona** — сегодня ушло 7–10 тысяч кредитов на тесты (много)
- **Репо** — расхода кредитов нет совсем → после перехода масштабирование без ограничений
- **Лимит репо** — Петя сказал "без лимитов" — в него не упрёмся

#### GoSequence (00:00:12)
- Следующий шаг в пайплайне после проверки уникальностей
- Там хранятся и запускаются секвенсы
- Sofia уже смотрела секвенсы в GoSequence — нашла актуальные кампании

#### Механизм разметки / ярлыков (00:11:00)
