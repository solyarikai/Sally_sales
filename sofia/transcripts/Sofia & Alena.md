## 2. Заполнение таблицы расходов (Shareable Services)

### Общая структура

- У каждого SDR есть табличка расходов за месяц
- Расчетный период — примерно месяц, не нужно разделять по конкретным числам
- Нужно посмотреть, до какого числа сводила расходы предыдущий SDR (Анастасия), и продолжить со следующего дня
- Если есть ассистенты — прописать. Если не используешь — не нужно

### Как считать расходы по сервисам

- Есть отдельная табличка с расходами, куда вносятся все использованные сервисы
- Ищем сервис, прописываем, сколько потратили
- Apollo: в этом месяце подписку отключили, почти не использовали
- Find Emails: считается по количеству найденных email'ов. Каждый find email списывается. Если не нашлось email'ов — почти ничего не использовалось
- Cloud Code: начали использовать только с марта. Нужно уточнить у Пети биллинг за код, потому что там тоже есть usage

**Alena:** Я до сих пор использую Find Emails мануально, потому что Cloud Code ищет эти email'ы через API иногда очень долго. Он зависает на 2-3 часа, а если загружаешь напрямую в Find Emails, то ищет буквально 5 минут.

### Инфраструктура

- Инфраструктуру по почтам пишет Артем
- Это самый большой расход — зависит от количества почт
- У Alena 37 почт = $315/мес за Google Workspace
- Записывается, у кого сколько аккаунтов, общая сумма подписки делится на всех

### Компенсация расходов

- Бюджет на проект (например, $1800) — это деньги, которые даются на проект
- Задача — сделать затраты как можно меньше, остаток идет SDR
- Если оплатила с личной карты — в конце месяца компенсируют плюсом к зарплате
- Если с рабочей карточки — просто прикрепить платежку

**Alena:** Бывают классные месяцы, бывают нет. У меня часто не хватает на KPI лидов — KPI, например, 8, а было 7 лидов. Получается, KPI нет, получаю только фикс. Мне нужно, чтобы все остальное было покрыто и я не уходила в минус.

## 3. Маркетинговый анализ и pipeline

### Рынок

**Sofia:** Я сделала параллельный маркетинговый анализ. Весь рынок маркетинговых агентств, платформ, куда нужна эта API-шка — проблема в том, что чуваки продают API. Это не все маркетинговые агентства мира, а именно те, у которых platform-based, или in-house platform, или есть какой-то движок, или техническая команда, которые не сделали эту API сами и готовы платить за нее.

Сам рынок огромный — 26 биллионов долларов, а сам рынок потребителей именно этого продукта маленький. Нейросеть говорит 500 компаний. Плюс-минус такая емкость.

### Pipeline обработки

1. Собрали 70 тысяч компаний широким фильтром из Apollo
2. Прогнали через blacklist по Magnum Opus — осталась половина
3. Скрапинг сайтов + GPT-классификация (6-я версия промта)
4. Код перепроверяет после GPT
5. Sofia выборочно проверяет вручную на маленьком батче
6. Запуск на всю базу
7. Результат: чистая таблица на 3-5 тысяч компаний

### Проблемы с GPT

- Корреляции между ключевыми словами Apollo и реальным содержанием сайта нет — компании не пишут о себе "SaaS", "platform", "marketing", а описывают свой продукт другими словами
- GPT ленится: если сайт не открывается — все равно проставляет сегмент и кладет в "таргет"
- Поэтому три этапа перепроверки

**Sofia:** В промте есть интересная штука: "если будет очень много повторяющихся платформ сегмента, который мы еще не придумали — придумай сегмент и собери их в кучку". Каждый день формируются новые сегменты рынка.

## 4. Расширение воронки — способы поиска лидов

### 4.1. Внешние базы данных

**Alena:** Иногда в общем доступе есть открытые базы, которые кто-то уже собрал. Я на GigaSheet выкачала базу по fashion brands на 20 тысяч контактов бесплатно. Отфильтровали, прогнали через GPT — получили 15 заинтересованных лидов и 5-6 звонков в прошлом месяце.

Хотя казалось, что мы эту базу уже должны были затрагивать — оказалось, что что-то не попало в Apollo, что-то раньше отсеивали по другим фильтрам, а сейчас промт поменялся.

### 4.2. Конференции

- Найти конференции, подходящие под ICP
- Спарсить списки экcгибиторов (участников-выставщиков)
- Спарсить участников через LinkedIn-группы ивентов
- Если нет списка участников — спарсить лайки и комментарии под постами конференции, убрать дубли
- Loom: бывают онлайн-ивенты, участников легко спарсить
- LinkedIn-ивенты: посмотреть участников, убрать безработных

**Важно:** Если парсишь камерные конференции — лучше не ссылаться на конференцию в письме. Организаторы могут пожаловаться. У Alena такое было.

### 4.3. LinkedIn-группы

- Найти группы по интересам (максимально узкие)
- Спарсить через SalesNav
- Не брать слишком широкие группы (100К+ контактов)

### 4.4. Расширение людей из компании

**Alena:** Сколько сейчас берется из компании людей?

**Sofia:** Изначально брала только супер-таргетов (CTO, CO), потом просто оставила всех heads, C-level, убрала exclude тех, кто не подходит, остальных всех фигачим. Если даже это не он — есть шанс, что перешлет письмо коллеге.

### 4.5. General emails

- Спарсить общие почты (hello@, info@) через Cloud Code
- Исключить: customer support
- Подходят: info-почты, hello-почты
- Они перенаправляются секретарем тому сотруднику, который принимает решения
- При этом письмо приходит БЕЗ плашки "External" — больше доверия

### 4.6. Переиспользование базы

**Sofia:** Я прогоняю каждый раз через blacklist даже тех, с кем добавляла за последние три месяца. И даже так у меня все равно остаются процентов 60 новых компаний, хотя Apollo сто раз переиспользовался.

**Alena:** Тебе не обязательно всегда искать новые компании. Если проект будет долго — можешь просто взять домены, которые раньше были, и по ним найти новых людей — кого-то уволили, кого-то наняли.

## 5. Работа с клиентами (фаундерами)

### Расширение ICP

**Sofia:** Клиенты говорят "нам нужны компании от 20 сотрудников", дали лист типа Patreon — крупняки. Но я понимаю, что современные AI-driven компании могут иметь 5 человек и серьезные деньги. Они устарели в своем ICP. Я не спорю — запускаю как гипотезы, подсыпаю небольшие компании.

Вчера пришел лид — три фаундера, маленькая команда — оказался SQL. Результат доказывает, что клиенты заблуждаются.

### Как презентовать расширение сегментов

**Alena:** Спокойно объясняешь: большие компании — их немного на рынке, цикл сделки 6-12 месяцев, нужно заутричить 20-30 человек из разных отделов. Давайте подстрахуемся — запустим компании, где принятие решений быстрее.

- План А: таргетные крупные компании, спокойно аутричить, напоминать
- Остальное: расширить воронку на компании до 20 человек, где до фаундера можно достучаться

**Alena:** Не забывай, что их основная цель — заработать больше денег. Любую идею подводи к этому: "эти компании быстрее принесут вам деньги". Язык денег, а не звонков.

### Проблема: клиенты сами аутричат

**Sofia:** Они начали на первой же встрече спрашивать, какие компании хорошо работают. Потом: "можешь дать лиды, которые загостились, мы их дожмем". На следующий день кто-то из них в моей таблице тусуется — а там листы с verified email'ами.

**Alena:** Веди себе отдельно листы. Вставляй ссылку в статистику только когда кампания запущена — тогда они увидят контакты, но уже после запуска.

### Проблема: собственники не приходят на звонки

**Alena:** У меня основной фаундер ушел в отпуск на 2 недели в январе. Запасной фаундер проигнорил все 4 звонка. Половина лидов больше не согласилась на встречу. Просто прекратили назначать в январе — один SQL за месяц.

## 8. Региональная стратегия

**Alena:** США отвечает меньше. Я бы миксовала: США — 20%, остальное — Европа. Можно затронуть русскоговорящие страны (Казахстан, Узбекистан) — наши на русском языке отвечают больше. Чеки не суперклассные, но явно больше, чем в Индии.

**По Индии:** Там быстро будет идти (сразу три заинтересованных за день), но чеки небольшие. Средний чек, который индусы платят индусам — намного меньше. Могут посчитать стоимость внедрения API и собрать то же самое с помощью разработчиков за $200/мес.

**По России:** Много блокировок, зарубежное ПО попадает под ограничения. Даже Claude без VPN не работает. Не таргетим.

## 12. Оптимизация работы с Cloud Code

**Alena:** У меня токены на Pro закончились через час активной работы. Тратит очень много на Find Emails. Гораздо проще:
1. Загрузить лист в Find Emails мануально (3 минуты)
2. Скачать результат
3. Отдать готовый файл с email'ами в Cloud Code
4. Он берет результаты и продолжает работать — токенов тратит меньше

### Три способа поиска через Find Emails
1. Company domain + first/last name
2. LinkedIn URL
3. Домены напрямую

Cloud Code прогоняет по всем трем — поэтому долго. Мануально быстрее для больших батчей.
