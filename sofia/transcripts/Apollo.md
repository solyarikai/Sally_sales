# Созвон по Apollo: фильтры, blacklist, пайплайн сегментации

**Участники:** Соня, Амир
**Тема:** Настройка фильтров Apollo для выгрузки 20 000 контактов (сегмент Influencer Platforms)

---

## 1. Начальная задача

Нужно собрать 20 000 контактов из Apollo. Три сегмента, самый большой — Influencer Platforms. Начинаем с настройки фильтров.

## 2. Какие фильтры в Apollo работают плохо

- **Market Segments** — сильно режет выборку, лучше скипать
- **Buying Intent** — плохо работает
- **Инвестиционные раунды** — Apollo плохо подтягивает эту информацию
- **Revenue** — многие компании попадают в "ноль", потому что данных нет. Лучше брать максимально широко

## 3. Exclusion Lists и Blacklist

### Компании vs Люди

- Работа с компаниями в Apollo **бесплатна** — можно выгружать, загружать, создавать списки без траты кредитов
- Загрузка людей **тратит кредиты** на enrich почтами

### Как загружать exclusion list компаний

1. Зайти в Companies → Import → CSV
2. **Матчить только по сайту** (один столбец с доменом). Если добавить название компании, Apollo может плохо распознать
3. Продублировать вкладку перед загрузкой, чтобы не потерять фильтры

### Blacklist из Smartlead (6000 контактов)

- Можно исключить на уровне людей
- Лучше также исключить на уровне компаний: отфильтровать уникальные домены, убрать дубликаты через Google Sheets, загрузить как exclusion list компаний
- **Нюанс:** при долгих проектах (полгода+) нельзя полностью исключать компании — могут появиться новые ЛПР, которые не попадут в выборку

### Матчинг людей в exclusion list

- Лучше всего матчить по имени (email)
- Если по людям работает плохо — сделать на уровне компании

## 4. Лайфхаки Apollo

### Сохранение фильтров

- Скопировать URL с применёнными фильтрами — при вставке все фильтры восстанавливаются
- Apollo чувствительный к кликам: не нажимать на название индустрии — нажимать на "+45" (количество), иначе фильтры сбрасываются

### Lookalike

- В Apollo есть функция lookalike
- В Clay lookalike удобнее: загрузила 80 компаний, три итерации — база выросла в 2 раза

### Исключение по выдаче (практический подход)

1. Выставить фильтры
2. Посмотреть, какие компании выдаёт
3. Открыть сайты через столбец Links
4. Исключать нерелевантное: e-commerce, sportswear, online shop, gambling
5. Повторять, пока исключения не начнут резать полезную выборку

## 5. Apollo vs Clay

| Параметр | Apollo | Clay |
|----------|--------|------|
| База | SalesNav под капотом, обновляется периодически | SalesNav в реальном времени |
| Ёмкость | Плюс-минус одинаковая | Плюс-минус одинаковая |
| Exclusion list | Нет ограничений на компании | Лимит 1000 строк на таблицу |
| Интерфейс | Чувствительный, часто сбрасывает фильтры | Современнее, стабильнее |
| Стоимость | — | Нужны кредиты |
| Lookalike | Есть | Удобнее |

## 6. Дедупликация

Лучший способ — **на уровне таблиц по очищенному домену:**

1. Попросить Claude/AI формулу, которая из домена делает очищенный домен (убирает http/https, www, слэши)
2. Использовать VLOOKUP для исключения пересечений
3. Это надёжнее, чем пытаться дедуплицировать через нейросеть

## 7. Пайплайн сегментации через Crona

### Правильный подход к промптам (итеративная валидация)

1. Выгрузить компании из Apollo
2. Загрузить в Crona
3. **Запустить промпт на 25 компаниях** (не на всём объёме!)
4. Выгрузить результат, протыкать каждый сайт вручную
5. Создать три столбца: сайт | как промпт определил | как на самом деле
6. Отдать эту таблицу обратно в Claude Opus с просьбой улучшить промпт
7. Повторять, пока точность не станет приемлемой
8. Запустить на полном объёме (10 000+)

> Этот подход сохранится и после миграции с Crona на Claude Code.

## 8. Воронка и объёмы

- Конверсия из грязных контактов Apollo в готовых к аутричу: **~30%** (1 к 3)
- Для 20 000 готовых контактов нужно выгрузить **~60 000** из Apollo
- Фильтровать лучше на уровне компаний — это дешевле (одна компания = одна строка анализа, а не 3-5 строк по людям)

## 9. Стратегия по локации

- Клиент хочет лиды из любых стран
- Приоритет: United States, United Kingdom (денежные клиенты)
- Не ставить жёсткие ограничения по локации — расширить воронку на входе, сегментировать потом

## 10. Crunchbase

- Хороший источник для стартапов: точные данные по слияниям, инвестициям, фандингу
- Амир использует для венчурного фонда (сбор стартапов, маленькие объёмы, важна точность)
