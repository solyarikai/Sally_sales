## Содержание

Представьте ситуацию: в вашей базе данных есть дубликаты, и вы хотите оставить только уникальные строки. Просто выделите таблицу, перейдите в **Данные** и **Очистка данных**. Затем нажмите **Удалить дубликаты**. Здесь вы выбираете столбцы, которые нужно проанализировать.

## Удаление дубликатов

Например, вы хотите удалить полностью одинаковые строки — когда все столбцы в этих строках совпадают. Просто нажмите "Выбрать все". Посмотрите на пример: Mira Shah дублируется три раза — строки абсолютно идентичны. Нажимаем "Удалить дубликаты" — и все лишние дублирующиеся строки удалены.

## Использование фильтров

Ещё один важный инструмент — фильтры. Фильтры позволяют создать нужный вам вид данных. Например, вы хотите таргетировать только SaaS-компании — просто оставьте "SaaS" и посмотрите результат.

Или, например, вы *не* хотите таргетировать SaaS-компании. Отфильтруйте всю базу и затем удалите этих лидов. Важно: **не удаляйте содержимое ячеек** — нужно **удалить строку целиком**. Нажмите правой кнопкой и выберите "Удалить строку". После этого снимите фильтр. Видите — SaaS-компаний больше нет.

Нажимаем Ctrl+Z, чтобы отменить изменения.

## Исключение одного списка из другого через VLOOKUP

Ещё один важный сценарий использования функции VLOOKUP — когда нужно исключить один список из другого. Представьте, что у вас очень большой список компаний, и вы хотите убедиться, что он не пересекается с другим списком.

В этом случае: выделите компании, нажмите **Данные > Удалить дубликаты**. Допустим, у вас есть Amazon и Facebook. Нужно исключить из основного списка те компании, которые есть во втором списке.

Используем функцию VLOOKUP: выбираем столбец поиска, выбираем диапазон (фиксируем знаками доллара), выбираем только первый столбец. Пишем `0`, как всегда. Видно, что "Acorn Labs" есть в этом списке. А Google, Amazon, Facebook — в другом списке их нет, поэтому получаем `#N/A`.

После этого легко применить фильтр — отфильтровать строки с `#N/A` и удалить эти компании. Снимите фильтр — список очищен. У вас остались только компании, которые существуют в нужном списке.

Это и есть сценарий исключения одного списка из другого.
