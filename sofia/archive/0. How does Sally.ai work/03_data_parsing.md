# Парсинг и сбор B2B-данных — источники и инструменты

## Что такое парсинг в контексте B2B-продаж

Парсинг (data parsing/scraping) — это автоматизированный сбор данных из различных источников для формирования базы потенциальных клиентов. В B2B-аутриче это первый технический шаг после определения ICP: нужно найти конкретные компании и людей, которые соответствуют профилю.

## Источник 1: Apollo.io

### Что это
AI-платформа для продаж. Крупнейшая открытая база B2B-контактов: 210M+ контактов и 30M+ компаний. Объединяет базу данных, аутрич, CRM и аналитику в одном продукте.

### Ключевые функции для парсинга

**Поиск компаний и людей:**
- Фильтры по индустрии, размеру, выручке, географии, технологиям
- Фильтр по должностям: CEO, VP of Sales, Head of Growth и т.д.
- Buyer intent signals — сигналы покупательского намерения

**Lookalike (AI-поиск похожих компаний):**
- Указываешь 1–3 компании-образца, которые идеально подходят под ICP
- AI находит похожие компании по характеристикам
- Можно дополнительно фильтровать по индустрии, ключевым словам, размеру
- Решает проблему слишком общих ключевых слов, которые выдавали нецелевые компании

**Market Segments (бета-фильтр):**
- Фильтр по типу рынка/бизнес-модели: B2B, SaaS, E-commerce, Fintech, D2C
- Раньше фильтр "Software Development" мог выдать и B2B SaaS, и разработчика игр, и аутсорс-студию
- Теперь можно отсекать по бизнес-модели сразу

**Верифицированные контакты:**
- Email-адреса и телефоны с верификацией
- Данные обновляются регулярно
- Встроенная валидация

### Как используют в Sally
1. Стартовый источник данных — первый шаг в waterfall enrichment
2. Lookalike для поиска компаний, похожих на лучших клиентов
3. Market Segments для отсечения нецелевых бизнес-моделей
4. Комбинация с Crona для дообогащения данных

## Источник 2: LinkedIn Sales Navigator

### Что это
Премиум-инструмент LinkedIn для поиска и отслеживания B2B-лидов. Стоит от $79.99/мес.

### Ключевые возможности
- **Расширенные фильтры:** по компании, должности, стажу, активности, взаимным связям
- **Lead & Account lists:** сохранение и организация найденных контактов
- **InMail:** прямые сообщения людям вне вашей сети
- **Alerts:** уведомления о смене должности, публикациях лидов
- **Boolean Search:** сложные поисковые запросы с операторами AND/OR/NOT

### Как используют в Sally
- Crona.ai имеет **Sales Navigator Agent** — собирает данные напрямую со страниц поиска Sales Navigator
- Используется как один из основных источников для сбора LinkedIn-профилей
- Далее обогащается рабочими email через waterfall enrichment

## Источник 3: Конференции и ивенты

### Почему это важно
Конференции — один из самых конверсионных источников лидов для Sally. Участники конференций — активные профессионалы в своих нишах, что даёт высокий intent signal.

### Метод Sally: парсинг приложений конференций

**Шаг 1: Найти веб-приложение конференции**
Большинство конференций используют конструкторы (Brella.io, Grip, Swapcard). Даже если на сайте указаны только мобильные приложения:
1. Найти приложение в App Store
2. Посмотреть блок "Copyright" — часто указан конструктор
3. Нажать "Developer's Privacy Policy" — ведёт на сайт конструктора
4. Зайти на сайт конструктора и авторизоваться с данными билета

**Шаг 2: Парсинг через Crona Scraper (Chrome Extension)**
- Заходите в веб-версию приложения конференции
- Запускаете парсер в 2 клика
- Он автоматически собирает всех участников (10–20k контактов на крупной конференции)

**Шаг 3: Обогащение в Crona**
- LinkedIn-профили
- Рабочие email
- Сегментация по ICP через AI

**Шаг 4: Аутрич ДО конференции**
- Запускать кампании за несколько недель до ивента
- "Будем на ICE. Давайте встретимся?" 
- Результат: забитый календарь + ни минуты зря на стенде

### Кейс: Gateway Crypto + ICE Barcelona
- 10–20k контактов на одну конференцию
- 3 когорты ICP: iGaming operators, iGaming platforms, payment orchestration
- Аутрич через email, LinkedIn и приложение конференции
- После конференции — follow-up: "Не успели встретиться. Созвонимся онлайн?"

## Источник 4: Telegram-чаты

### Зачем
Telegram — основной источник контактов для русскоговорящего рынка, Web3, iGaming.

### Как работает парсинг Telegram через Crona
1. Зайти в web.telegram.org/a (только эта версия поддерживается)
2. Открыть нужный чат
3. Запустить Crona Scraper extension
4. Парсер собирает публичные данные участников
5. Нажать "Enrich with Crona" → попадаете в проект с обогащением
6. Добавляете LinkedIn, рабочие email, сайты компаний
7. Фильтруете через AI, оставляя только ICP

**Ограничение:** собираются только данные, открытые в настройках профиля. Если номер скрыт — не соберётся.

## Источник 5: Дополнительные инструменты

### Apify
- Платформа для веб-скрапинга с готовыми "актёрами" (скриптами) для сотен сайтов
- Используется Sally для парсинга Google Maps, Yelp, специализированных директорий
- Даёт CSV-списки с телефонами и сайтами — конверсия с этих источников 4%+

### Octoparse
- No-code инструмент для веб-скрапинга
- Визуальный интерфейс: указываешь элементы на странице → инструмент собирает данные
- Используется для сбора данных конференций

### BrowserFlow
- Автоматизация действий в браузере
- Используется для автоматического аутрича внутри веб-приложений конференций
- Можно настроить автоматическую отправку сообщений участникам

## Иерархия источников данных в Sally

```
1. Конференции (Crona Scraper)     — высший intent, горячие лиды
2. Sales Navigator (Crona Agent)    — точный таргетинг по ICP
3. Apollo.io (Lookalike + Segments) — масштаб, широкий охват
4. Telegram-чаты (Crona Extension)  — нишевые сообщества
5. Google Maps / Yelp / Apify       — локальный бизнес, SMB
6. CSV Upload в Crona               — свои данные из любого источника
```

## Ключевой принцип

Данные из разных источников имеют разную ценность. Участник конференции, который купил билет за $2,000 — это сигнал высокого intent. Контакт из Apollo без сигналов — это холодный лид. Sally всегда начинает с источников с наивысшим intent и спускается вниз по иерархии.
