## 2. РОЛЬ И МОДЕЛЬ ОПЛАТЫ

### Что такое Project Owner
Ты не наёмный сотрудник с зарплатой. Ты **внутренний предприниматель**: владеешь бюджетом проекта, отвечаешь за результат (SQL), и забираешь прибыль.

**SQL = Sales Qualified Lead** — лид, который соответствует ICP, ответил на аутрич и готов к встрече.

### Финансовая модель

**Базовый сценарий (1 проект):**
- Бюджет: $1,800/мес
- Расходы на сервисы: до $200
- KPI первого месяца: 6 SQL
- Чистый доход: ~$1,600

**Бонусы:**
- За каждый SQL выше 10: +$150
- 20 SQL = $1,800 - $200 + (10 × $150) = ~$3,100

**Масштабирование:**
- После 10 SQL/проект — можешь нанимать ассистентов
- 2 проекта × 2 ассистента × 20 SQL каждый = бюджет $8,000, расходы ~$1,600, чистыми ~$6,400
- При 50+ SQL/мес — делишься с клиентами revenue, получаешь % от сделок за первый год

## 3. ПОЛНЫЙ ПРОЦЕСС: ОТ ICP ДО SQL

Это главное, что нужно знать. Весь процесс лидгена в Sally состоит из 6 шагов.

### Шаг 1: Определение ICP (Ideal Customer Profile)

ICP — описание идеальной компании-клиента по 4 измерениям:
- **Firmographics:** индустрия, размер, выручка, география
- **Technographics:** какие технологии/инструменты используют
- **Organizational:** структура команды, стадия роста
- **Behavioral:** нанимают ли на нужные позиции, участвуют ли в конференциях, получали ли инвестиции

**Пример ICP из кейса Sally:** Не просто "финтех-компания", а "B2B финтех, 50+ сотрудников, работает с крипто-платежами, имеет API, ищет expansion в EU, участвует в конференциях iGaming-индустрии."

**Принцип:** ICP ≠ индустрия. ICP = конкретный набор характеристик, по которым можно отфильтровать базу.

### Шаг 2: Сегментация

Широкий ICP разбивается на узкие сегменты — каждому свой оффер.

**Как это делает Sally:**
- Загружают список компаний в Crona.ai
- Описывают словами критерии: "раздели на iGaming operators, iGaming platforms и payment orchestration"
- AI анализирует сайт каждой компании и распределяет по категориям

**Пример (кейс INXY Payments):**
- Широкий ICP: криптокомпании, SaaS, гейминг, маркетплейсы
- Сегменты: 50+ персонализированных связок "сегмент + оффер"
- Дополнительный слой: уже работают с криптой vs нет → разные офферы
- Результат: 15–20+ квал встреч/мес

### Шаг 3: Парсинг базы (сбор данных)

Источники данных, от самых горячих к холодным:

| Источник | Как собирают | Intent |
|----------|-------------|--------|
| Конференции | Crona Scraper (Chrome extension) — парсинг веб-приложения конференции | Высший |
| Sales Navigator | Crona Sales Navigator Agent | Высокий |
| Apollo.io | Lookalike + Market Segments фильтры | Средний |
| Telegram-чаты | Crona extension для web.telegram.org/a | Нишевый |
| Директории, Google Maps | Apify, Octoparse | Базовый |

**Конференции — главный козырь Sally.** Парсят 10–20k участников через Chrome extension, сегментируют, запускают аутрич ЗА НЕСКОЛЬКО НЕДЕЛЬ ДО конференции. К моменту ивента — календарь забит встречами.

### Шаг 4: Энричмент (обогащение данных)

После парсинга у тебя есть имена и компании, но нет контактов. Энричмент добавляет: рабочий email, телефон, LinkedIn, данные о компании.

**Waterfall enrichment** — ключевая концепция: запрос идёт последовательно через несколько провайдеров, пока не найдёт данные.

```
Apollo (дешёвый) → Hunter.io → Findymail → Dropcontact → ...
          │                          │
          └─── email найден? ────────┘
                     │
                     ▼
              NeverBounce (валидация)
                     │
                     ▼
              Готов к аутричу ✓
```

**Почему waterfall:** один провайдер покрывает 60–75%. Waterfall доводит до 90–95%.

**Основные инструменты:**
- **Hunter.io** — находит email по имени + домену. 6M+ пользователей. Строит паттерны (firstname.lastname@domain) и проверяет через SMTP
- **Findymail** — AI-powered email finder, находит на 10–30% больше чем конкуренты, гарантия <5% bounce
- **NeverBounce** — валидатор, проверяет каждый email до 75 раз, 20+ шагов верификации

**Валидация email работает в 3 этапа:**
1. Syntax check (формат адреса)
2. DNS lookup (существует ли домен, есть ли MX-записи)
3. SMTP handshake (сервер подтверждает существование ящика без отправки письма)

### Шаг 5: Аутрич (отправка)

**Каналы:**
- Email (основной) — через Instantly или Smartlead
- LinkedIn — коннект + серия сообщений
- WhatsApp/Telegram — для высоко-intent нишь (крипто, iGaming)

**Инфраструктура email:**
- Домены-расходники (не основной домен!)
- Настройка SPF, DKIM, DMARC
- Прогрев 3–4 месяца перед полной нагрузкой
- Reply Rate в warmup: 80–90%
- Unlimited email accounts в Instantly/Smartlead с автоматической ротацией

**Instantly.ai** — платформа для cold email: unlimited ящиков, встроенный warmup, AI Spintax, Unibox (единый inbox).

**Smartlead.ai** — аналог с фокусом на доставляемость: выделенная инфраструктура, собственные IP, SmartAgents.

### Шаг 6: Сиквенс, квалификация и booking

**Структура сиквенса: Pain → Value → Proof → CTA**

| Элемент | Что это | Пример |
|---------|---------|--------|
| Pain | Конкретная проблема получателя | "При 50+ cold emails/день без автоматизации bounce rate сжигает домены за 2 месяца" |
| Value | Конкретная ценность прямо сейчас | "Хочешь, пришлём сэмпл данных под твой проект?" |
| Proof | Кейс/цифра | "Для INXY генерируем 20+ квал встреч/мес" |
| CTA | Одно конкретное действие | "Ответьте 'yes' и пришлю" |

**Time-to-value:** ценность должна быть в ПЕРВОМ касании. Не оффер, не "давайте созвонимся", а конкретная польза: данные, анализ, инсайт.

**Длина email:** до 80 слов. Суть за 5 секунд.

**Сиквенс из 4 emails:**
1. Day 0: Pain + Value + Proof + CTA
2. Day 3–4: Другой угол (другая pain point или use case)
3. Day 7–8: Social proof (кейс подробнее)
4. Day 14: Breakup ("Понимаю, сейчас не приоритет")

**Квалификация ответов:**
- "Не интересно" → закрыть уважительно
- "Расскажите подробнее" → квалифицировать по ICP
- "Давайте созвонимся" → SQL! Booking через Calendly
