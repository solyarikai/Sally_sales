## 2026-03-05 — Segmentation prompt (Deliryo → ArchiStruct)

**Паттерны из оригинала Deliryo:**
- Чёткая роль в начале: "You are helping X identify companies that could partner with Y"
- Сегменты с описанием — не просто название, а кто это и почему релевантен
- Примеры вывода для каждого сегмента — модель видит эталон
- Секция `## Input` в конце — куда Crona подставляет данные

**Что добавила команда ArchiStruct (улучшения):**
- Явные if→then правила: "If company builds luxury villas → VILLA_DEVELOPERS" — убирают двусмысленность
- Positive signals + Negative signals — список географии, терминов, антипаттернов
- Строгий формат: "One word only" на строке 1 — критично для парсинга в Crona
- Edge case явно прописан: "If website is inactive → OTHER + note"

**Вывод:** Описательные инструкции слабее явных правил. Всегда добавлять if→then + signals.

## 2026-03-05 — Email body prompt (Deliryo)

**Паттерны:**
- Шаблоны по сегментам — один промпт, несколько веток по типу компании
- Переменные в фигурных скобках: `{property type: villas/apartments}` — дефолт прямо в скобках
- Строгие запреты в конце: NO signatures, NO formatting, NO additional content
- Один CTA в конце каждого шаблона — всегда вопрос о звонке

**Вывод:** Для генерации писем — шаблоны с переменными эффективнее чем "написать письмо для X". Модель не импровизирует, а заполняет форму.

## 2026-03-05 — Subject line prompt (Deliryo)

**Паттерны:**
- Максимум 50 символов — ограничение заставляет быть точным
- Валюты в CAPS (USDT, BTC) — конкретное правило форматирования
- NO восклицательных знаков, NO ALL CAPS — запреты на спам-триггеры
- Один шаблон на сегмент, минимум переменных

**Вывод:** Subject line промпты должны иметь жёсткие символьные ограничения и список запретов.

## 2026-03-05 — Cold email structure (Royal Moving Co пример)

**Структура эффективного холодного письма:**
1. Subject = конкретный оффер (не вопрос, не интрига)
2. Параграф 1 = боль → решение (одно предложение)
3. Параграф 2 = кейс с цифрами + имя реального клиента + неожиданный факт + результат
4. CTA = низкобарьерный ("10 минут") + конкретное действие
5. Подпись = имя + компания + 2–3 известных клиента

**Ключевое:** Конкретные цифры (127 предметов, 22 вендора, 11 ошибок) делают кейс убедительным.
