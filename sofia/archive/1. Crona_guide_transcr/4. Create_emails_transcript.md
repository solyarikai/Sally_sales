## Filtering Out Non-Fit Companies

As you remember, we have created a new column containing the sector. Our main goal now is to filter out the companies that were marked as "Other" — because we have specific segments: real estate agencies, real estate developers, proptech, fintech platforms, and "other." These companies are not a fit and we want to delete them.

To do that, we add a filter. A filter basically deletes rows using a rule. Here you can see the segmentation enricher name — I'll rename this enricher and call it "segment." After the result, we add our enricher name and delete companies named "Other."

In real life you'll have a lot of companies marked as "Other" and we want to delete them. This is the same as downloading the results to Google Sheets and manually filtering and deleting rows you don't need.

## Creating Personalized Emails

Now we have all the companies that are a fit for us. And now we want to create emails for these companies. So we add another enricher — select AI, copy my already prepared prompt for creating emails, and paste it.

What this enricher does: it creates email bodies for my client depending on the **sector** and depending on the **website**. So I have a two-level personalization:
1. **By sector** — each sector has its own email template
2. **By website content** — the email is adapted based on what's on the company's website

## Email Template Structure

For example:
- **Real estate developers** → one specific template
- **Real estate agencies** → another template
- **Legal finance services** → another template

Each sector has its own template.

Also an important rule: I use the company's name as a variable and I want to clean it up. So I have a rule that cleans the company's name — I want the output to sound clean, without any caps lock, symbols, etc.

## Additional Personalization Variables

- **Property type** — I analyze the website and depending on the property type, the prompt selects one of several types.
- **Geographic operations / location** — it will depend on the website's location and target market.

Strict requirements: only the template. We need only the clean email. Maintain the structure. No signatures, no text formatting, no additional content. Output format: email body text only — no headers, labels, or explanations.

## Input Data

The input data we provide:
- **Website** — the company's website URL
- **Website text** — the scraped website content
- **Sector / Segment** — the segment column from our data

## Results

As you can see, we have ready emails. For example, agencies will have this one: *"Your Russian clients are increasingly asking about cryptocurrency payment options — become the new standard. Delirio provides fully legal USDT & BTC acceptance for real estate transactions, Cyprus..."* And legal/financial services will have another template.

## Key Takeaways

- Remember that you'll need to **create a template for each sector** — that is really important.
- You can take the other rules from my example (strict requirements, etc.) and adapt the emails using location, currencies.
- For example, you can analyze the website and determine which country the company is from, then use the currencies from those countries in your emails.

You have a huge variety of ways to tailor your emails — but the most important thing is to create templates for each sector and to use the website, website text, and sector as input columns.
