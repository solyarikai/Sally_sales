## 3. BEST IDEAS (украденные друг у друга)

Идеи, которые другие команды явно назвали "они правы, а мы это упустили". Это рекомендации с максимальным сигналом -- их одобрили ЧУЖИЕ рецензенты.

### 3.1 Сегментация по покупательскому поведению, а не по типу компании (BRAVO)
**Кто украл:** Alpha, Charlie, Delta -- все трое в Round 2 признали превосходство этой модели.

Текущая сегментация: "Platforms vs Agencies vs PR firms" -- по типу компании.
Bravo предлагает: **"API Buyers vs Tool Buyers vs Enterprise Strategic"** -- по тому, КАК они покупают.

- **API Buyers** (CTO/VP Eng) -- строят продукт, им нужен data pipeline. Им продаём API, интеграцию, white-label.
- **Tool Buyers** (CEO/Head of IM) -- им нужен дашборд, отчёты, визуализация. Им НЕ нужны API endpoints.
- **Enterprise Strategic** (Kantar, Spotter, Jellysmack) -- ABM, персональный подход, 3-6 месяцев цикл.

**Почему это лучше:** Brighter Click (агентство) ведёт себя как API Buyer. BrandNation (SaaS) ведёт себя как Tool Buyer. Тип компании не определяет, как они покупают -- определяет наличие инженерной команды и тип задачи.

### 3.2 "Внутренний монолог покупателя" и fear-of-loss фрейминг (DELTA)
**Кто украл:** Alpha (10/10 за Copywriter), Bravo (9/10), Charlie (9/10)

Delta читает каждый email глазами CTO в 9:14 утра: "This person doesn't know me. They're pitching. Delete." Это не стилистика -- это метод, который обнажает проблемы, невидимые из позиции отправителя.

Fear-of-loss rewrite для агентств: "Агентство потеряло крупного клиента, потому что рекомендовало 3 блогеров с 60% overlap аудитории" -- эмоциональный триггер сильнее, чем "сэкономьте 6 часов".

### 3.3 AFFPERF кампания -- доказанный сегмент без выделенной кампании (ALPHA)

impact.com -- strong fit -- буквально affiliate/performance платформа. Сотни таких платформ (Partnerize, Awin, CJ Affiliate, Rakuten, ShareASale, Refersion, Everflow, PartnerStack, Tune) не тронуты. AFFPERF framework уже существует в документе сиквенсов, но никогда не был запущен.

### 3.5 Unit economics по сегментам и SDR time audit (CHARLIE)
**Кто украл:** Alpha (9/10 за Analyst), Bravo, Delta

Charlie -- единственная команда, рассчитавшая:
- Cost per reply: PR firms $75 vs Marketing agencies $7.24 (разница в 10x!)
- SDR time на LinkedIn: 568 invites x 2 мин = 19 часов работы = 0 встреч = 0 дохода
- Экономия от автоматизации: 6.5ч/нед -> 2.5ч/нед = 4ч/нед = 2 рабочих дня в месяц

### 3.6 Валидация social proof как юридический/репутационный риск (CHARLIE)
**Кто украл:** Alpha (включил в Quick Wins), Bravo (признал как критическое упущение)

Вопрос: "Modash, Captiv8, Kolsquare, Influencity, Phyllo, Lefty" -- это РЕАЛЬНЫЕ paying customers? Если нет, заявление "all run on our API" -- это бомба для репутации. Enterprise покупатели проверяют такие claims. Опции:
- Если клиенты: оставить и расширять
- Если trial/free: сменить формулировку на "have tested" или "have integrated"
- Если не клиенты: убрать немедленно

## 4. CONTESTED (команды расходятся -- нужно решение человека)

### 4.1 PR Firms: убить, поставить на паузу или микро-тест?

| Позиция | Команды | Аргумент |
|---------|---------|----------|
| **Убить полностью** | Delta, Bravo (lean) | 0.2% reply rate, 0 встреч, 0 interested из ~1000 контактов. Каждый отправленный email -- это сожжённый лид, который никогда не получит второго шанса. Плюс ущерб domain reputation. |
| **Паузить + переписать + микро-тест 100-200** | Alpha (10%), Charlie (pause) | 0.2% отражает ПЛОХОЙ КОПИ, а не мёртвый сегмент. NeoReach, Buttermilk -- это не PR-фирмы, social proof нерелевантен. Пока не тестировали правильный messaging. |

**Аргумент за микро-тест:** Сиквенс для PR-фирм использует social proof от IM-платформ (NeoReach, Buttermilk, Gushcloud) -- это полностью нерелевантно для PR. Все 4 команды согласны, что messaging сломан. Нельзя объявить сегмент мёртвым, если ни разу не тестировали с правильным посылом.

**Аргумент за полное закрытие:** Даже при "правильном" 1.2% (242 sent), 0 interested и 0 meetings. Объём контактов конечен -- каждый сожжённый PR-контакт невозвратен. Лучше перенаправить на 6.2%-сегмент.

**РЕШЕНИЕ АРБИТРА:** Паузить немедленно. Переписать сиквенс (см. раздел Sequence Rewrites). Микро-тест 200 контактов с НОВЫМ messaging через 2 недели. Если reply rate < 1.0% через 2 недели -- закрыть сегмент окончательно. НЕ отправлять ни одного контакта на текущем сиквенсе.

| Позиция | Команды |

### Структура follow-up писем (Steps 2-4) для всех сегментов

**Источник:** Delta (единственная команда, предложившая framework для всей последовательности)

**Текущая проблема:** Step 1 = pitch. Step 2 = тот же pitch, другие слова. Step 3 = тот же pitch + больше деталей. Покупатель, открывший Step 2 после игнора Step 1, получает НОЛЬ новой информации.

| Step | Цель | Формат | Пример для Platforms |
|------|------|--------|---------------------|
