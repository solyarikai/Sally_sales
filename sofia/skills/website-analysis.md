# Website Analysis

## Инструкции

1. Читай `projects/[project]/README.md` и `.claude/memory/MEMORY.md` — контекст проекта
2. Если задача завязана на конкретный проект, проверь `.claude/memory/feedback/project_file_context.md`, чтобы не перепутать папки проектов
3. Открой сайт через доступный веб-инструмент текущего рантайма
4. При необходимости просмотри `About`, `Services`, `Cases`, `Pricing`, `FAQ`
5. Для внешних сигналов используй доступный поисковый инструмент (SerpAPI, web search, firecrawl или аналог)
6. Если `analysis.md` уже существует, обнови его, а не создавай параллельную версию
7. Сохрани результат в `projects/[project]/analysis.md`

## Формат анализа

### Компания
- Название, локация, рынок

### Продукт / Услуга
- Что именно продают, ключевые услуги

### Ценностное предложение
- Главный оффер (1-2 предложения)
- Доказательства: цифры, кейсы, сроки

### Целевая аудитория
- Кто покупает, сегменты

### Боли клиентов
- Что решает продукт

### Выводы для аутрича
- На что делать акцент в письмах
- Приоритетный сегмент
- Персонализация: триггеры, хуки
