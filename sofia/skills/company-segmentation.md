# Скилл: Company Segmentation

Классификация компаний по сегментам на основе реального контента их сайтов. НЕ доверяем Apollo-фильтрам — только визит на сайт.

## Когда использовать

Пользователь говорит:
- "классифицируй компании"
- "сегментируй"
- "раскидай по сегментам"
- "проверь компании по сайтам"
- "кто из них нам подходит"

---

## Входные данные

Пользователь даёт:
1. **CSV с компаниями** — из Apollo или другого источника
2. **Промпт с описанием сегментов** — какие сегменты, критерии попадания, примеры

Если промпта нет — спросить. Сегменты каждый раз разные, зависят от проекта и ICP.

---

## Процесс

### Шаг 1 — Подготовка: получить сайты компаний

Если в CSV нет колонки Website — сначала запустить скилл `enrich-contacts.md` (шаг 2: поиск сайтов).

Результат: каждая компания имеет URL сайта.

---

### Шаг 2 — Фетчинг контента сайтов

Параллельно скачать контент всех сайтов. Это ключевой шаг — без него классификация будет гадание.

```python
import aiohttp, asyncio, json
from bs4 import BeautifulSoup

async def fetch_one(session, sem, item):
    url = item.get('website', '')
    if not url:
        item['fetch_status'] = 'no_url'
        item['content'] = {}
        return item
    async with sem:
        try:
            async with session.get(url, timeout=aiohttp.ClientTimeout(total=10),
                                   allow_redirects=True, ssl=False) as r:
                html = await r.text()
                item['content'] = extract_text(html)
                item['fetch_status'] = 'ok'
        except Exception as e:
            item['fetch_status'] = str(type(e).__name__)
            item['content'] = {}
    return item

async def fetch_all(items, concurrency=20):
    sem = asyncio.Semaphore(concurrency)
    async with aiohttp.ClientSession(
        headers={'User-Agent': 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)'}
    ) as session:
        tasks = [fetch_one(session, sem, item) for item in items]
        return await asyncio.gather(*tasks)

def extract_text(html):
    soup = BeautifulSoup(html, 'html.parser')
    for tag in soup(['script','style','nav','footer']): tag.decompose()
    return {
        'title': soup.title.get_text(strip=True) if soup.title else '',
        'meta_desc': (soup.find('meta', attrs={'name': 'description'}) or {}).get('content', '')[:300],
        'h1': ' | '.join(h.get_text(strip=True) for h in soup.find_all('h1')[:3]),
        'body': ' '.join(soup.get_text().split())[:1200],
    }
```

**Кэширование**: сохранять результат в `/tmp/website_content.json`. Если файл есть — не перефетчивать.

**Ожидаемый результат**: ~70% сайтов отдадут контент. Остальные — таймауты, редиректы, CloudFlare.

---

### Шаг 3 — Классификация (я и есть LLM)

**КРИТИЧНО: НЕ использовать эвристики, keyword-matching или regex. Читать контент сайта и классифицировать как LLM.**

#### Порядок:

1. **Дедупликация** — составить список уникальных компаний (по названию). Контактов может быть 400, компаний — 100-200.

2. **Батчинг** — обрабатывать компании батчами по 20-30. Для каждой компании смотреть:
   - Website title + meta description
   - H1 заголовки
   - Body excerpt (первые 1200 символов)
   - Apollo industries / keywords (как подсказка, НЕ как истина)
   - Количество сотрудников

3. **Классификация** — для каждой компании определить сегмент + написать reason (1 предложение, почему именно этот сегмент).

4. **Компании без контента** — если сайт не загрузился:
   - Попробовать определить по названию компании + Apollo данным
   - Если недостаточно данных → пометить `NEEDS_REVIEW`

#### Что НЕ делать:
- НЕ доверять Apollo industries/keywords слепо (~78% ошибок в нашем опыте)
- НЕ использовать keyword-matching ("if 'influencer' in title → INFLUENCER")
- НЕ классифицировать по домену или названию без проверки контента

---

### Шаг 4 — Проверка качества

После классификации:

1. **Показать распределение** — сколько компаний в каждом сегменте
2. **Показать спорные кейсы** — компании с неочевидной классификацией
3. **Показать NEEDS_REVIEW** — компании без достаточных данных
4. **Spot check** — вывести 5-10 примеров из каждого сегмента для проверки пользователем

```
=== Распределение ===
SEGMENT_A: 35
SEGMENT_B: 12
SEGMENT_C: 50
OTHER: 352

=== Примеры SEGMENT_A ===
• Company X — SaaS platform for creator analytics
• Company Y — Influencer discovery tool with API
...
```

Пользователь может попросить пересмотреть конкретные компании — пересмотреть.

---

### Шаг 5 — Выходные файлы

Добавить к исходному CSV две колонки:
- `Segment` — название сегмента
- `Segment_reason` — причина классификации (1 предложение)

Разделить на файлы:

```
segments/[дата]/
├── raw_export.csv                    # оригинал (не трогать)
├── Segment_classified.csv            # полный файл с колонкой Segment
├── [Segment]_for_smartlead.csv       # контакты целевых сегментов
└── other_review.csv                  # контакты сегмента OTHER для ручной проверки
```

## Оптимизация

### Скорость
- Фетчинг: 20 параллельных запросов, ~200 сайтов за 30 секунд
- Классификация: батчи по 20-30, весь процесс ~5 минут на 400 компаний

### Качество
- Всегда показывать примеры пользователю перед финализацией
- При сомнении — лучше OTHER, чем неправильный сегмент
- Крупные компании (1000+ сотрудников) обычно не целевые
- Если компания — бренд/клиент, а не инструмент/агентство → OTHER

### Кэширование
- `/tmp/website_content.json` — контент сайтов (не перефетчивать)
- `/tmp/classification_output.json` — результаты классификации (для повторного использования)

## Оптимальный пайплайн (проверено март 2026)

### Фетчинг: двухэтапный

1. **Этап 1 — aiohttp** (бесплатно): параллельный фетчинг всех сайтов. ~70% отдадут контент.
2. **Этап 2 — Crona** (6 кр/компания): для компаний без контента (SSL-ошибки, CloudFlare, нет URL). Crona делает google_search (находит сайт) + scrape_website (вытаскивает контент через свои прокси).

### Классификация: только Claude в контексте

**НЕ использовать Crona call_ai (GPT-4o-mini) для классификации.**

Тестировано на 83 компаниях (март 2026):
- GPT-4o-mini: 45% precision (лучший результат после 3 итераций промпта)
- Claude в контексте: ~97% precision

Причина: GPT-4o-mini галлюцинирует "likely influencer" на пустых данных и не может отличить IM-first от IM-as-side-service по одной строке Google description.

Промпты для Crona call_ai сохранены в `starter-kit/segmentation_prompt_4o_mini.md` (v2) — можно использовать как pre-filter, но НЕ как финальную классификацию.

## Уроки из практики

**Март 2026 — OnSocial классификация:**
- Apollo фильтры говорили "influencer marketing" → **78% оказались OTHER** после проверки сайтов
- Эвристики нашли 8 из 35 реальных платформ → переход на LLM-классификацию
- Финальный результат: 449 компаний, 97 целевых (22%), 352 OTHER
- **Мораль: никаких эвристик, только чтение сайта**

**Тестирование промпта для GPT-4o-mini (Crona call_ai):**
- v1 (длинный промпт): 14% precision — модель игнорирует правила
- v2 (compact + hard rules): 45% precision — правила работают, но потолок данных
- v3 (stricter): 45% precision — улучшение невозможно без контента сайта
- Вывод: Crona = фетчинг, Claude = классификация
- Полный eval log: `starter-kit/segmentation_prompt_eval.md`

## Примечания

- `python3.11` — использовать для фетчинга (aiohttp установлен)
- Сегменты задаются промптом от пользователя, НЕ захардкожены
- Всегда сохранять reason — чтобы можно было проверить логику
- При большом количестве компаний (500+) — фетчить и классифицировать порциями
