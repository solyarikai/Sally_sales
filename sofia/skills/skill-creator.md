# Skill Creator

## Процесс

**Шаг 1 — Понять скилл**

Спроси у пользователя (если не очевидно из контекста):
1. Что должен делать скилл?
2. Когда Claude должен его применять — какие фразы или ситуации триггерят?

**Шаг 2 — Создать файл**

Создай `skills/[name].md` с такой структурой:

```
---
name: [строчные буквы и дефисы, до 64 символов]
description: [что делает + когда использовать, до 1024 символов]
---

# Название

## Инструкции

[Краткие шаги. Не объясняй очевидное — Claude уже умный.]

## Примеры

[Пары ввод → вывод, если формат вывода важен]
```

**Шаг 3 — Подтвердить**

Скажи: "Скилл '[name]' создан в `skills/`. Триггер: [когда]."

## Правила хорошего скилла

- `description` — главное: именно по нему Claude решает применять ли скилл
- Пиши от третьего лица в description ("Создаёт...", не "Я создаю...")
- Не объясняй что Claude и так знает
- Если скилл большой — основной файл содержит только обзор, детали в отдельных файлах рядом
- Пути к файлам проекта: `projects/[project]/`, скиллы: `skills/`

## Пример

Ввод: "Сохрани как скилл: перед отправкой письма всегда проверяй тон через SerpAPI"

Вывод — файл `skills/email-tone-check.md`:
```
---
name: email-tone-check
description: Проверяет тон холодного письма перед отправкой. Используй когда пользователь говорит "проверь письмо", "готово ли письмо", "можно отправлять".
---

# Email Tone Check

## Инструкции

1. Прочитай письмо
2. Проверь: нет ли давления, есть ли персонализация, понятна ли ценность
3. Дай оценку и 1-2 конкретных правки если нужно
```
